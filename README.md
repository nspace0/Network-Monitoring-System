# Network Monitoring System

**Network Monitoring System** — консольное приложение на **C++17** для автоматизации проверки сетевого оборудования.

Программа предназначена для системных администраторов и позволяет хранить список устройств в JSON-файле, автоматически проверять их доступность и в дальнейшем получать информацию по различным сетевым протоколам.

---

# Возможности

На текущий момент реализовано:

- ✅ Парсинг JSON-конфигурации
- ✅ Загрузка списка устройств
- ✅ Проверка корректности конфигурационного файла
- ✅ Проверка TCP-портов:
  - SSH (22)
  - Telnet (23)
  - HTTP (80)
  - HTTPS (443)
  - SNMP (161)
- ✅ Сборка проекта через CMake

Планируется реализовать:

- ICMP Ping
- Измерение задержки (Latency)
- Traceroute
- Многопоточное сканирование
- SSH-подключение
- Telnet-подключение
- HTTP/HTTPS запросы
- Получение информации по SNMP
- Формирование отчётов
- Логирование
- Экспорт результатов (CSV / HTML / JSON)

# Сборка проекта

Перейдите в корневую папку проекта.

Создайте директорию сборки:

```bash
mkdir build
```

Перейдите в неё:

```bash
cd build
```

Сгенерируйте проект:

```bash
cmake ..
```

Соберите проект:

```bash
cmake --build .
```

После успешной сборки появится исполняемый файл:

```text
build/network_scanner
```

---

# Запуск

Запуск рекомендуется выполнять **из корневой директории проекта**.

Пример:

```bash
./build/network_scanner
```

# Конфигурационный файл

Конфигурация устройств хранится в:

```text
config/devices.json
```

Пример структуры:

```json
{
    "devices": [
        {
            "name": "CoreSwitch",
            "type": "Коммутатор",
            "location": "Серверная",
            "model": "SNR-S2990G",
            "ip": "10.90.90.90",
            "mac": "00:00:00:00:00:0",
            "vlan": 404,

            "credentials":
            {
                "login": "admin",
                "password": "admin"
            }
        }
    ]
}
```
